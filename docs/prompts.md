## 2. Agent Orchestration State Machine Instructions

The core agent operates following the formal loops detailed below:

```text
IDLE -> PLANNING -> EXECUTING -> VERIFYING -> REPORTING -> COMPLETED/FAILED
```

And utilizes the full cycle:

```text
Observe -> Reason -> Plan -> Execute -> Verify -> Learn -> Report
```

### Prompt for Error Correction State Transition:
```text
State: VERIFY_FAILED
Scenario: Drill Step Promotions are locked in Read-Only Configuration.
Agent Instruction:
1. READ step output and logs.
2. DETERMINE underlying failover constraint.
3. TRIGGER standard CLI commands or target script parameter updates to unlock mount paths.
4. RETRY execution. If retries exceed 3, transite state to FAILED and escalate notifications to PagerDuty dashboard.
```
